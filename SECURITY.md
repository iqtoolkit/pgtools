# Security Policy

The `pgtools` project takes security seriously. We appreciate your efforts to responsibly disclose your findings, and will make every effort to acknowledge your contributions.

## Supported Versions

Use this section to tell people about which versions of your project are

## Reporting a Vulnerability

Use this section to tell people how to report a vulnerability.
Please report any security vulnerabilities privately to ensure the security of our users. **Do not create a public GitHub issue.**

Tell them where to go, how often they can expect to get an update on a
reported vulnerability, what to expect if the vulnerability is accepted or
declined, etc.
We support and encourage private disclosure through GitHub Security Advisories.

You can expect the following process:
1.  We will acknowledge your report within 3 business days.
2.  We will provide a more detailed response within 7 business days, indicating the next steps in handling your report.
3.  We will keep you informed of our progress as we investigate and (if applicable) patch the vulnerability.
4.  You will be publicly credited for your discovery once the vulnerability is disclosed.
